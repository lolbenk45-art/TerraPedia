# Summary Templates

## Lightweight Stage Summary

Use for most coding and audit tasks.

```md
# <topic>

## Goal
- <single current goal>

## Scope
- Files: <file whitelist>
- Excluded: <main exclusions>

## Facts
- <confirmed fact 1>
- <confirmed fact 2>

## Changes
- <change made>

## Validation
- <test or verification result>

## Next Step
- <next concrete action>
```

## Handoff Summary

Use when the task will continue in a later round.

```md
# <topic> handoff

## Current State
- <what is done>

## Important Files
- <path 1>
- <path 2>

## Open Risks
- <risk 1>

## Next Action
- <one next action>
```

## Audit Summary

Use for data and report-heavy work.

```md
# <audit topic>

## Question
- <what was audited>

## Sources Used
- <summary md or report json>

## Findings
- <finding 1>
- <finding 2>

## Evidence
- <short reference only>

## Recommendation
- <next action>
```
