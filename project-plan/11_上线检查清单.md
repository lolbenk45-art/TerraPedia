## 3. 上线日操作流程

1. **代码合并与部署**
   - [ ] 确认所有改动已提交到 Git
   - [ ] 合并到 main 分支
   - [ ] 触发 Cloudflare Pages 部署（或等待自动部署）
   - [ ] 在 Cloudflare Dashboard 确认部署状态为 Success

2. **DNS 与访问验证**
   - [ ] 检查生产域名解析是否指向正确
   - [ ] 使用 `nslookup` 或在线工具验证 DNS 生效
   - [ ] 在无痕模式下访问网站，确认无缓存干扰

3. **功能抽检**
   - [ ] 访问首页
   - [ ] 随机抽查 3–5 个核心页面
   - [ ] 测试搜索功能
   - [ ] 在手机端快速验证响应式

4. **监控确认**
   - [ ] 确认 Analytics 有数据上报
   - [ ] 确认 Search Console 可抓取
   - [ ] 确认可用性监控显示在线

5. **宣布上线**
   - [ ] 发布上线公告
   - [ ] 保存部署 commit hash 或版本号，便于回滚

## 4. 回滚预案

### 触发条件

- [ ] 核心功能大面积不可用（如全站 500、白屏）
- [ ] 发现严重安全漏洞
- [ ] 数据错误导致内容严重失真
- [ ] 性能严重劣化（首屏 > 5s 或 Lighthouse < 50）

### 回滚步骤

1. **Cloudflare Pages 回滚**
   - [ ] 登录 Cloudflare Dashboard → Pages → 项目
   - [ ] 进入 Deployments 页面
   - [ ] 找到上一个稳定版本，点击「Rollback to this deployment」
   - [ ] 确认回滚完成

2. **或 Git 回滚后重新部署**
   - [ ] 本地执行 `git revert` 或 `git reset` 回退到稳定 commit
   - [ ] 强制推送到 main（谨慎操作）
   - [ ] 等待 Cloudflare 自动重新部署

3. **回滚后处理**
   - [ ] 验证回滚后网站可正常访问
   - [ ] 在本地或分支中排查问题原因
   - [ ] 修复后重新部署前再次走完整检查流程
   - [ ] 记录事故原因与处理过程，便于复盘
