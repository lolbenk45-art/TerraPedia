# 技术选型

## 方案对比总览

针对游戏 Wiki 类网站，主要有三条技术路线可选。下面从多个维度进行对比分析。

---

## 路线一：静态站点生成器（SSG）

### 候选方案

| 框架 | 语言 | 特点 |
|------|------|------|
| **Astro** | JS/TS | 岛屿架构，按需加载 JS，多框架兼容 |
| VitePress | JS/TS | 基于 Vue，Markdown 驱动，极致简洁 |
| Docusaurus | JS/TS | React 生态，文档型站点首选 |
| Hugo | Go | 构建速度最快，模板语法学习曲线陡 |

### 优势

- **极致性能**：纯静态 HTML，首屏加载极快，天然 SEO 友好
- **零运维成本**：部署到 Vercel / Netlify / Cloudflare Pages，免费额度足够
- **安全性高**：无服务端、无数据库，攻击面极小
- **CDN 全球分发**：静态资源天然适合 CDN
- **版本控制**：内容即代码，全部在 Git 中管理

### 劣势

- **动态功能受限**：搜索、用户系统等需要额外方案
- **构建时间**：5000+ 页面时构建可能需要数分钟
- **内容更新延迟**：每次修改需要重新构建部署

### 适用场景

内容相对稳定、以阅读为主、不需要用户实时交互的 Wiki 站点。

---

## 路线二：全栈框架（SSR/ISR）

### 候选方案

| 框架 | 语言 | 特点 |
|------|------|------|
| **Next.js** | JS/TS | React 生态，SSR/SSG/ISR 灵活切换 |
| Nuxt.js | JS/TS | Vue 生态，同样支持多种渲染模式 |
| SvelteKit | JS/TS | Svelte 生态，编译时优化，包体积小 |

### 优势

- **渲染策略灵活**：静态页面用 SSG，搜索页面用 SSR，最佳混合方案
- **全栈能力**：API Routes 内置，无需独立后端
- **增量静态再生（ISR）**：内容更新无需全量重建
- **丰富生态**：组件库、状态管理、Auth 方案完善

### 劣势

- **托管成本**：需要 Node.js 运行时，免费额度有限
- **复杂度高**：框架概念多，对 Solo 开发者负担更大
- **服务端维护**：有运行时就有宕机风险，需要监控

### 适用场景

需要动态搜索、用户系统、API 服务等服务端能力的项目。

---

## 路线三：Wiki 引擎

### 候选方案

| 引擎 | 语言 | 特点 |
|------|------|------|
| **Wiki.js** | Node.js | 现代化 UI，Markdown 编辑，Git 同步 |
| MediaWiki | PHP | 维基百科同款，功能最完善，社区最大 |
| BookStack | PHP | 书籍结构组织，简洁易用 |

### 优势

- **开箱即用**：编辑器、权限、搜索、多语言内置
- **社区协作**：天然支持多人编辑、版本历史、审核流程
- **MediaWiki 生态**：大量模板和插件，游戏 Wiki 社区经验丰富

### 劣势

- **性能一般**：服务端渲染，SEO 需要额外配置
- **定制成本高**：想要自定义 UI 需要深入了解模板系统
- **服务器成本**：需要 VPS / 云服务器，月费 $5-20
- **维护负担**：数据库备份、安全更新、系统运维

### 适用场景

内容量巨大且需要多人协作编辑的社区驱动 Wiki。

## 推荐方案

### 首选：Astro（静态站点生成器）

**理由：**

1. **Solo 开发者最优解**：零运维、零费用、性能天花板
2. **数据驱动架构**：将游戏数据存为 JSON/YAML，Astro Content Collections 原生支持
3. **岛屿架构**：静态页面中按需插入交互组件（搜索框、筛选器），不牺牲性能
4. **渐进增强**：MVP 纯静态，后续可集成 React/Vue 组件添加动态功能
5. **部署极简**：推送到 GitHub 自动部署到 Cloudflare Pages，全球 CDN

### 搜索方案：Pagefind

- 构建时生成搜索索引，纯客户端搜索
- 无需后端服务，搜索响应 < 100ms
- 支持中文分词

### 后备方案：Next.js

如果后续需要以下功能，可迁移到 Next.js：
- 用户账号系统
- 实时 API 服务
- 服务端搜索（数据量超过 Pagefind 上限）

## 配套技术栈

### 前端

| 组件 | 选型 | 说明 |
|------|------|------|
| 框架 | Astro 5.x | SSG 核心 |
| UI 组件 | Astro 原生 + 少量 React | 岛屿架构按需加载 |
| 样式方案 | Tailwind CSS 4.x | 原子化 CSS，开发效率高 |
| 搜索 | Pagefind | 构建时索引，客户端搜索 |
| 图标 | Iconify / Lucide | SVG 图标按需加载 |

### 数据层

| 组件 | 选型 | 说明 |
|------|------|------|
| 数据格式 | YAML / JSON | 结构化游戏数据 |
| 内容管理 | Astro Content Collections | 类型安全的内容层 |
| 图片处理 | Astro Image / Sharp | 自动 WebP 转换 + 响应式图片 |

### 部署与基础设施

| 组件 | 选型 | 说明 |
|------|------|------|
| 代码托管 | GitHub | 版本控制 + CI/CD 触发 |
| CI/CD | GitHub Actions | 自动构建 + 部署 |
| 托管平台 | Cloudflare Pages | 免费、全球 CDN、自动 HTTPS |
| 域名 | Cloudflare DNS | 统一管理，自带 DDoS 防护 |
| 图片 CDN | Cloudflare R2 (可选) | 大量图片资源存储 |

### 开发工具

| 工具 | 用途 |
|------|------|
| VS Code / Cursor | 代码编辑器 |
| pnpm | 包管理器（比 npm 快，磁盘占用小） |
| ESLint + Prettier | 代码规范与格式化 |
| TypeScript | 类型安全 |
| Husky + lint-staged | Git 钩子，提交前自动检查 |
