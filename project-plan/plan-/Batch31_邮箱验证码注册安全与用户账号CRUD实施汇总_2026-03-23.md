# Batch31 邮箱验证码注册安全与用户账号 CRUD 实施汇总（2026-03-23）

## 1. 本轮目标
- 新增“邮箱验证码注册”能力，注册必须校验验证码。
- 增加防刷、防撞库安全策略。
- 用户端支持对自己账号进行 CRUD（创建/读取/更新/删除）。
- 优化登录/注册/账号中心 UI。

## 2. 后端实现

### 2.1 注册邮箱验证码能力
- 新增接口：`POST /api/user-auth/register/code`
- 新增请求 DTO：`UserRegisterCodeRequestDTO`
- 新增响应 DTO：`UserRegisterCodeResponseDTO`
- 注册接口改造：`POST /api/user-auth/register`
  - `UserRegisterRequestDTO` 新增 `verificationCode`
  - 注册流程新增验证码校验，不再允许“仅邮箱+密码”注册

### 2.2 邮件发送能力（SMTP）
- 引入依赖：`spring-boot-starter-mail`
- 新增配置类：`MailProperties`（`terraria.mail.*`）
- 新增服务：`RegisterVerificationService`
  - 生成 4~8 位数字验证码（默认 6 位）
  - 通过 `spring.mail` 发信
  - 验证码哈希存储到 Redis（不存明文）

### 2.3 安全策略（防刷/防撞库）
- 登录防撞库增强（`LoginRateLimitService`）：
  - 原“邮箱+IP”维度保留
  - 新增“邮箱维度”失败计数/锁定
  - 新增“IP 维度”失败计数/锁定
- 注册验证码防刷（`RegisterVerificationService`）：
  - 发送冷却（默认 60s）
  - 每邮箱每小时发送上限
  - 每 IP 每小时发送上限
  - 验证失败计数与临时锁定

### 2.4 用户端账号 CRUD（自助）
- 保留：创建（Register）、读取（`GET /api/user-auth/me`）
- 新增：
  - `PATCH /api/user-auth/profile` 更新显示名
  - `PATCH /api/user-auth/password` 修改密码
  - `DELETE /api/user-auth/account` 注销/删除账户（逻辑删除）
- 鉴权拦截器更新：上述接口已纳入用户鉴权保护。
- 注销账户/改密后会撤销 refresh token。

### 2.5 Cookie 安全可配置
- `UserAuthProperties` 新增：
  - `cookieSecure`
  - `cookieSameSite`
- 登录态 Cookie 不再写死安全属性，改为配置驱动。

## 3. 前端实现

### 3.1 注册流程升级
- `front/src/api/userAuth.ts` 新增：
  - `sendRegisterCode`
  - `updateUserProfile`
  - `changeUserPassword`
  - `deleteOwnAccount`
- `front/src/stores/userAuth.ts` 新增：
  - `requestRegisterCode`
  - `updateProfile`
  - `changePassword`
  - `deleteAccount`
- `UserRegisterView.vue`：
  - 增加验证码输入框
  - 增加“发送验证码”按钮与倒计时
  - 提示验证码有效期

### 3.2 用户账号自助管理页
- `UserProfileView.vue` 重构为账号中心：
  - 查看账号信息
  - 修改显示名
  - 修改密码
  - 删除账户（需当前密码 + 二次确认）

### 3.3 UI 优化
- `UserLoginView.vue`、`UserRegisterView.vue` 采用更清晰的双栏/信息卡布局
- 增强视觉层次、提示信息与移动端适配
