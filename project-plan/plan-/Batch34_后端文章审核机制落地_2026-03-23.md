# Batch34 后端文章编写与审核机制落地（2026-03-23）

## 已完成
- 实现四态审核流：`DRAFT -> PENDING_REVIEW -> APPROVED/REJECTED`。
- 发布/下线与审核解耦：只有 `APPROVED` 允许发布。
- 新增审核日志表与接口，可追踪每次流转。
- 兼容旧接口 `PATCH /admin/articles/{id}/status`：
  - 默认兼容直发（`enforce-strict=false`）
  - 自动补 `APPROVED` 并记录 `DIRECT_PUBLISH_COMPAT`。

## 新增/调整接口（后端）
- `POST /api/admin/articles/{id}/submit-review`
- `POST /api/admin/articles/{id}/review`（`action=APPROVE|REJECT`, `comment`）
- `POST /api/admin/articles/{id}/publish`
- `POST /api/admin/articles/{id}/offline`
- `GET /api/admin/articles/{id}/review-logs?page=1&limit=20`
- 兼容旧接口：`PATCH /api/admin/articles/{id}/status`
