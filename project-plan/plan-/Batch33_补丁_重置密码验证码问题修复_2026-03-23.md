# Batch33-补丁  重置密码验证码二次校验问题修复（2026-03-23）

## 问题现象
- 忘记密码页输入正确验证码后，仍提示：`Verification code is expired or not requested`。

## 根因分析
- 原流程中，重置密码接口先验验证码再查用户。
- 当邮箱不存在/已删除时，验证码会被先消费，后续再次提交就会出现“expired or not requested”。
- 另外联调脚本默认会删除新建测试账号，容易让测试邮箱变成“未注册”，放大该问题。

## 修复内容
1. 后端流程修复（避免验证码无谓消耗）
- 文件：`back/src/main/java/com/terraria/skills/service/impl/UserAuthServiceImpl.java`
- 调整：
  - `sendPasswordResetVerificationCode` 先校验邮箱是否为已注册且激活账号；否则直接报错。
  - `resetPasswordByVerificationCode` 先校验用户存在且激活，再执行验证码校验。
- 新错误语义：
  - 未注册邮箱：`Email is not registered`
  - 禁用账号：`Account is disabled`

2. 前端提示优化
- 文件：`front/src/views/UserForgotPasswordView.vue`
- 调整：
  - 对 `Email is not registered` 做友好提示：`This email is not registered yet. Please register first.`

3. 联调脚本安全修复
- 文件：`scripts/dev/test_batch31_user_security_flow.py`
- 调整：
  - 默认不删除新建测试账号，避免误删造成后续重置密码失败。
  - 若确需删除，显式设置：`BATCH31_ALLOW_DELETE_TEMP_USER=true`。

## 验证结果
- `cd back && mvn -q test`：通过
- `cd front && pnpm run check`：通过
- `python scripts/dev/test_batch31_user_security_flow.py`：通过（`success=true`）
  - 新增/关键项通过：
    - `Password reset code send`
    - `Password reset by email code`
    - `User login (email-reset password)`

## 你现在可以这样操作
1. 若邮箱未注册：先走注册流程。
2. 若已注册：在忘记密码页点击 `Send Code` 后，使用最新一封邮件里的验证码提交。
3. 若还报错，把当次请求时间点发我，我直接按日志键定位（验证码 key / Redis TTL / 接口返回）。
