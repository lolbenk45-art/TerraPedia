# 需求分析

## 用户画像

### 主要用户群体

| 用户类型 | 描述 | 典型场景 | 占比（估） |
|----------|------|----------|-----------|
| 新手玩家 | 刚接触泰拉瑞亚，需要基础引导 | "铁矿在哪里挖？""怎么召唤第一个 Boss？" | 35% |
| 中期玩家 | 熟悉基础，需要合成配方与进阶攻略 | "泰拉刃怎么合成？""哪个配饰组合最优？" | 40% |
| 硬核玩家 | 追求极限数据、速通、建筑 | "各武器 DPS 对比""Boss 掉落概率表" | 15% |
| 模组/开发者 | 需要原始数据做二次开发 | "导出物品 ID 列表""API 接口调用" | 10% |

### 用户核心诉求

1. **快速找到信息** —— 3 秒内定位到目标内容
2. **数据准确可信** —— 与游戏实际数据一致
3. **移动端可用** —— 游戏中随时查阅
4. **无干扰阅读** —— 无广告、排版清晰

## 功能需求

### P0: MVP 必须功能

#### F01 物品数据库

- 展示所有物品的基础属性（名称、图标、类型、稀有度、描述、卖价）
- 物品详情页：完整属性、获取方式、使用场景
- 支持按类型分类浏览（武器、工具、盔甲、配饰、消耗品、材料等）

#### F02 搜索系统

- 全局搜索框，支持中英文物品名称搜索
- 搜索联想/自动补全
- 搜索结果高亮匹配关键词

#### F03 分类导航

- 顶部/侧边导航栏：物品、NPC、Boss、生态群落、事件
- 面包屑导航
- 分类层级：大类 > 子类 > 具体条目

#### F04 响应式布局

- 桌面端：三栏布局（导航 + 内容 + 侧边栏）
- 平板：两栏布局
- 手机：单栏布局 + 底部导航

#### F05 基础 SEO

- 语义化 HTML 结构
- 每个页面独立的 title / description / og 标签
- 自动生成 sitemap.xml
- robots.txt 配置

### P1: V1.0 增强功能

#### F06 合成配方系统

- 可视化合成树（从原材料到最终产物的合成路径）
- 反向查询："这个材料能合成什么？"
- 制作站标注（工作台、铁砧、秘银砧等）

#### F07 Boss 攻略

- Boss 一览表（按进度排序）
- 每个 Boss 的详情页：属性、掉落物、召唤条件、战斗策略
- Boss 进度线（新手 → 困难模式 → 月亮领主）

#### F08 NPC 系统

- NPC 列表与详情（售卖物品、偏好生态、快乐值机制）
- NPC 住房需求可视化

#### F09 高级搜索与筛选

- 按稀有度、伤害类型、属性范围筛选
- 多条件组合筛选
- 排序功能（按伤害、防御、价格等）

#### F10 图片与资源

- 物品图标、Boss 立绘、生态群落缩略图
- 图片懒加载 + WebP 格式优化
- CDN 加速分发

### P2: V2.0 社区功能

#### F11 社区贡献

- 基于 GitHub PR 的内容贡献流程
- 贡献者排行榜
- 内容审核机制

#### F12 多语言支持

- 中文简体 / 繁体 / 英文
- URL 路径国际化（/zh/items/xxx, /en/items/xxx）

#### F13 暗黑模式

- 跟随系统偏好
- 手动切换开关
- 主题色持久化存储

#### F14 互动地图

- 泰拉瑞亚世界结构示意图
- 点击区域跳转至对应生态群落页面
- 标注关键资源/NPC 分布

#### F15 开放 API

- RESTful API 提供物品、NPC、Boss 数据查询
- API 文档（Swagger/OpenAPI）
- 速率限制与 API Key 管理

## 非功能需求

### 性能

| 指标 | MVP 目标 | V1.0 目标 |
|------|----------|-----------|
| 首屏加载 (LCP) | < 2.5s | < 1.5s |
| 首次输入延迟 (FID) | < 100ms | < 50ms |
| 累计布局偏移 (CLS) | < 0.1 | < 0.05 |
| 搜索响应时间 | < 500ms | < 200ms |
| 页面体积 (gzip) | < 200KB | < 150KB |

### SEO

- Google PageSpeed Insights 评分 > 90
- 所有内容页面可被搜索引擎索引
- 结构化数据标记（Schema.org）
- Open Graph + Twitter Card 支持

### 可用性

- 99.5% 以上可用率（基于静态托管）
- 全球 CDN 覆盖，任意地区访问延迟 < 500ms
- 支持 HTTP/2

### 可访问性

- WCAG 2.1 AA 级别合规
- 键盘导航支持
- 屏幕阅读器友好
- 色彩对比度 > 4.5:1

### 安全性

- HTTPS 强制启用
- CSP（Content Security Policy）头配置
- 无用户输入场景下的 XSS 防护
- 依赖项定期安全审计

## 用户故事

### 新手玩家

```
作为一名泰拉瑞亚新手，
我希望能通过搜索框输入"铁矿"，
快速看到铁矿的获取方式、用途和合成配方，
以便我知道下一步该做什么。
```

### 中期玩家

```
作为一名正在挑战肉山的玩家，
我希望能在 Boss 攻略页面看到肉山的召唤条件、属性和推荐装备，
以便我做好充分准备。
```

### 硬核玩家

```
作为一名追求最优配装的玩家，
我希望能按伤害类型和DPS排序所有武器，
以便我找到当前阶段的最强武器。
```

### 移动端用户

```
作为一名在游戏中需要随时查资料的玩家，
我希望在手机上也能流畅浏览和搜索，
不被广告遮挡，不需要横屏操作。
```

### 贡献者

```
作为一名热心社区成员，
我希望能通过简单的流程提交内容修正或新增，
并能看到我的贡献被记录和展示。
```
