# Batch30 实施与联调报告（2026-03-23）

## 1. 本批已落地范围

### A. 前台真实用户登录/注册
- 后端新增 `users`、`user_refresh_tokens` 数据模型与 Flyway 迁移。
- 新增用户认证接口：
  - `POST /api/user-auth/register`
  - `POST /api/user-auth/login`
  - `GET /api/user-auth/me`
  - `POST /api/user-auth/logout`
- 认证机制：Access Token + Refresh Token（Cookie: `HttpOnly + SameSite=Lax`）。
- 与后台管理员认证隔离（管理员继续使用 `/api/auth/*`）。

### B. 登录/注册保密与用户安全
- 密码存储改为 `BCrypt`（cost=12）。
- 登录限流上线（按 `IP + email`）：失败累积达到阈值后短时锁定。
- 安全审计日志表与记录机制上线（注册、登录成功/失败、登出、管理员重置密码等）。
- 登录失败统一返回，避免账号枚举。

### C. 后台用户管理模块
- 新增后台接口：
  - `GET /api/admin/users`
  - `PATCH /api/admin/users/{id}/status`
  - `POST /api/admin/users/{id}/reset-password`
- 管理端新增用户管理页（查询、启用/禁用、重置密码）。
- 禁用用户后前台登录会被拒绝。

### D. 文章管理模块（后台发布，前台只读）
- 后端新增 `articles` 模型与接口：
  - 后台：`/api/admin/articles*`（列表、详情、创建、编辑、状态流转）
  - 前台：`/api/articles`、`/api/articles/{id}`、`/api/articles/slug/{slug}`（仅发布态）
- 状态流转实现：`DRAFT / PUBLISHED / OFFLINE`。
- 管理端新增文章管理页。
- 前台新增文章列表页与详情页（Markdown 渲染）。

### E. 三端接入
- 前台（`front`）接入用户认证与文章模块：
  - 新增登录、注册、个人中心、文章列表、文章详情路由与页面。
  - 新增路由守卫（游客页/登录保护页）。
- 后台（`data-query-app`）新增菜单项：用户管理、文章管理。
