# 项目总览

## 项目定义

**TerraPedia** —— 一个现代化的泰拉瑞亚（Terraria）中文百科站点，为玩家提供结构化、可检索、视觉友好的游戏数据查询与攻略浏览体验。

### 核心差异化

| 维度 | Fandom Wiki | 本项目（TerraPedia） |
|------|------------|---------------------|
| 加载速度 | 重度广告拖慢、首屏 >5s | 静态优先，首屏 <2s |
| 移动端体验 | 广告遮挡、排版混乱 | 响应式设计，移动端优先 |
| 数据结构 | 非结构化 Wiki 文本 | 结构化 JSON/YAML 数据驱动 |
| 搜索体验 | 基础全文搜索 | 支持筛选、联想、分类搜索 |
| 中文支持 | 翻译不全、格式混乱 | 原生中文，术语统一 |
| 广告干扰 | 大量广告 | 无广告 |

---

## 项目愿景

打造泰拉瑞亚中文社区最权威、最快、体验最好的百科站点。让任何玩家在 3 秒内找到想要的游戏信息。

---

## 里程碑规划

> 以下时间估算基于 Solo 开发者（每天投入 2-4 小时）

### Phase 0: 规划与设计（第 1-2 周）

- [x] 需求分析与功能优先级排序
- [ ] 技术选型最终确认
- [ ] 架构设计完成
- [ ] UI/UX 原型设计（低保真线框图）
- [ ] 数据模型定义

### Phase 1: MVP（第 3-8 周）

**目标：核心功能可用，可公开访问**

- [ ] 项目脚手架搭建、开发环境配置
- [ ] 核心数据模型实现（物品、NPC、Boss）
- [ ] 物品数据库页面（列表 + 详情）
- [ ] 基础搜索功能
- [ ] 分类浏览导航
- [ ] 响应式布局适配
- [ ] 部署上线（域名 + HTTPS）
- [ ] 基础 SEO 配置

**交付标准：** 用户可以浏览和搜索泰拉瑞亚物品数据

### Phase 2: V1.0 正式版（第 9-16 周）

**目标：功能完善，内容丰富**

- [ ] 合成配方系统（可视化合成树）
- [ ] Boss 攻略专题页
- [ ] 高级搜索与筛选（按属性、稀有度、来源等）
- [ ] NPC 数据与住房需求
- [ ] 图片资源优化与 CDN 加速
- [ ] 站点地图与 SEO 深度优化
- [ ] 性能优化（Lighthouse 评分 > 90）
- [ ] 用户反馈渠道（GitHub Issues / 表单）

**交付标准：** 覆盖 80% 以上的核心游戏数据

### Phase 3: V2.0 社区版（第 17-24 周）

**目标：社区参与，持续增长**

- [ ] 社区贡献系统（GitHub PR 或在线编辑）
- [ ] 多语言支持（英文/繁体中文）
- [ ] 用户账号系统（可选）
- [ ] 暗黑模式
- [ ] 互动地图（世界生成预览）
- [ ] 数据版本管理（支持不同游戏版本）
- [ ] API 开放（供第三方应用调用）

## 风险识别

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| Solo 开发精力不足 | 高 | 高 | MVP 严格控制范围，优先自动化 |
| 数据录入工作量巨大 | 高 | 高 | 优先爬虫/脚本批量导入，再人工校验 |
| 游戏版本更新导致数据过时 | 中 | 中 | 建立版本化数据管理机制 |
| 图片版权问题 | 中 | 低 | 使用游戏社区通用素材，标注来源 |
| 服务器成本超预算 | 低 | 中 | 静态站点 + CDN，成本可控在 $5/月内 |
