# TerraPedia 本地数据源汇总（2026-03-28）

## 1. 真实本地数据源（主业务数据）

### 1.1 MySQL 本地库
后端当前支持两套本地数据库入口：

- `legacy` -> `terria_new`
- `v1-local` -> `terria_v1_local`

关键依据：

- `back/README.md`（数据库配置说明）
- `back/src/main/resources/application-legacy.yml`
- `back/src/main/resources/application-v1-local.yml`
- `back/src/main/resources/application.yml`（默认可由环境变量覆盖）

说明：

- 前台和管理端对 `/api` 的请求最终都走本地后端（默认 `localhost:8888`），后端再访问本地 MySQL。

## 2. 本地缓存数据源

### 2.1 Redis 本地实例
本地联调脚本会启动并使用 Redis：

- `127.0.0.1:6380`
- `requirepass root`

关键依据：

- `scripts/dev/start-local-stack.ps1`
- `scripts/dev/run-backend-v1-local.ps1`
- `back/src/main/resources/application-v1-local.yml`

说明：

- Redis 属于缓存与会话/限流等运行时数据源，不是主业务持久化源（主持久化仍是 MySQL）。

## 3. 本地 Mock 数据源（开发兜底/演示）

### 3.1 前台首页兜底 Mock
当前首页在 API 异常时会回退到本地常量：

- `mockItems`
- `mockCategories`

关键依据：

- `front/src/views/HomePage.vue`

### 3.2 管理端“数据查询工作台”内置 Mock API
查询页使用 `local-query` 路由返回内置示例数据，明确不写真实数据库。

关键依据：

- `data-query-app/pages/query.vue`
- `data-query-app/server/routes/local-query/tables.get.ts`
- `data-query-app/server/routes/local-query/items-structure.get.ts`
- `data-query-app/server/routes/local-query/query.post.ts`

说明：

- 该模块主要用于 SQL 工作台交互闭环验证，不代表真实 DB 查询链路。

## 4. 本地文件数据源（数据同步与导入链路）

### 4.1 外部抓取后落地到本地文件
抓取脚本将数据写入本地目录：

- 原始数据：`data/raw/wiki/*`
- 报告：`data/reports/fetch/*`

关键依据：

- `scripts/data/wiki-item-utils.mjs`（默认 API：`https://terraria.wiki.gg/api.php`）
- `scripts/data/fetch-wiki-iteminfo.mjs`

### 4.2 本地归一化文件
归一化脚本默认：

- 输入：`data/raw/wiki/module__iteminfo__data.latest.json`
- 输出：`data/normalized/items.wiki.json`

关键依据：

- `scripts/data/normalize-wiki-items.mjs`

### 4.3 本地导入到后端 API
导入脚本默认：

- 输入：`data/normalized/items.sample.json`（可自定义）
- 导入 URL：`http://localhost:8888/api/items/import`

关键依据：

- `scripts/data/import-items.mjs`
- `scripts/data/README.md`

说明：

- 这是“本地文件 -> 本地后端 API -> 本地 MySQL”的标准联调链路。

## 5. 请求路径与数据流（简版）

1. `front` / `data-query-app` 调用 `/api`
2. dev proxy 转发到 `localhost:8888`
3. 后端读取 MySQL（主）+ Redis（缓存）
4. API 失败时，前台部分页面回退到内置 mock
5. 数据同步场景下，脚本先把抓取结果落盘到本地 JSON，再导入后端

关键依据：

- `front/vite.config.ts`
- `data-query-app/nuxt.config.ts`
- `front/src/api/index.ts`

## 6. 结论

当前项目“本地数据源”并非单一来源，而是并行存在：

- 主业务：本地 MySQL
- 运行缓存：本地 Redis
- 开发兜底：前台/管理端内置 mock
- 数据治理：本地 JSON 文件（raw/normalized/report）

在联调或排查时，建议先确认当前是否处于：

- 真实库路径（`v1-local` / `legacy`）
- mock 路径（首页 fallback 或 `local-query`）
- 文件导入路径（`data/raw` -> `data/normalized` -> `import`）
