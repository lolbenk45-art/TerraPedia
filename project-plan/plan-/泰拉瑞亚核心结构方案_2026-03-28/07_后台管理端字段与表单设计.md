# 后台管理端字段与表单设计

## 1. 设计目标

后台不是给开发者看的表结构页，而是给维护数据的人用的操作界面。

因此表单设计要满足三点：

- 能快速录入
- 能稳定修改
- 能清楚维护关系数据

## 2. 物品管理页

- `id`
- `internal_name`
- `name_en`
- `name_zh`

- `rarity_id`

- `name_zh`

## 8. 图片管理设计

图片最好不要只放一个 URL 输入框。

建议支持：

- 主图选择
- 图片列表排序
- 上传
- 手动 URL
- 来源页
- 文件标题

每条图片关系最好能维护：

- `role`
- `provider`
- `source_file_title`
- `source_page`
- `cached_url`
- `is_primary`
- `sort_order`

## 9. 来源管理设计

`item_sources` 这类表单建议做成行编辑模式。

每行字段：

- `source_type`
- `source_ref_type`
- `source_ref_name`
- `quantity_text`
- `chance_text`
- `conditions`
- `notes`
- `biome_code`
- `sort_order`

如果这块做成纯文本大输入框，后续维护会很痛苦。

## 10. 后台 UI 结构建议

后台建议采用三栏或双栏布局：

- 左侧：实体列表与筛选
- 中间：基础表单
- 下方或右侧：关系数据编辑区

不要把所有字段堆在一个长表单里。

更合理的分区是：

- 基础信息
- 数值信息
- 图片
- 关系
- 来源与快照

## 11. 最终结论

后台管理端的关键不是“能改字段”，而是：

- 能维护复杂关系
- 能看清数据来源
- 能控制图片与排序
- 能避免维护人员把结构化数据写回备注文本
