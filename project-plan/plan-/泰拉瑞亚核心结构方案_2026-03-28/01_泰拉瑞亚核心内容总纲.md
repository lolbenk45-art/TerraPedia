# 泰拉瑞亚核心内容总纲

## 1. 核心定义

Terraria 的核心不是单一系统，而是一个互相咬合的循环：

- 探索世界
- 采集资源
- 合成装备与功能物
- 提升角色强度
- 挑战 Boss 与事件
- 解锁新阶段与新世界内容

如果压缩成一句话：

- Terraria 是一个以世界探索、资源采集、配装成长和阶段推进为核心的 2D 沙盒动作冒险游戏。

## 2. 最核心的系统

### 2.1 世界与探索

- 世界是程序生成的
- 地表、地下、洞穴、天空、地狱等区域共同构成推进舞台
- 地图结构直接影响资源、敌怪、掉落和推进速度

### 2.2 生物群落 `Biomes`

- 不同 biome 决定敌怪、材料、环境与掉落
- biome 会参与刷怪、钓鱼、事件、Boss 条件和资源获取
- 对 TerraPedia 来说，biome 必须是主数据域，不是附属标签

### 2.3 采集与合成

- Terraria 的装备升级高度依赖材料链与制作站
- 玩家通过资源采集、制作站解锁和配方链实现成长
- 因此 `recipes` 是和 `items` 同等级的重要域

### 2.4 角色成长

主要不是经验等级，而是系统性堆叠：

- `Health`
- `Mana`
- 武器
- 护甲
- 饰品
- Buff / Debuff

所以 Terraria 的强度核心是：

- 装备
- 属性上限
- 配装组合
- 临时状态效果

### 2.5 战斗、Boss 与事件

- Boss 是阶段门槛
- Event 是高强度资源与战斗场景
- 很多阶段推进都围绕 Boss 和 Event 展开

### 2.6 阶段推进 `Hardmode`

- Hardmode 不是单个机制，而是世界状态升级
- 进入 Hardmode 后，敌怪、资源、事件、Boss、掉落和配装路线都会变化
- 所以它必须被建模成“阶段系统”

### 2.7 NPC 与住房

- NPC 承担商店、服务、指引和功能解锁作用
- `House` 不是装饰规则，而是 NPC 生态系统的前提
- 住房条件和 NPC 入住会反过来影响玩家推进效率

### 2.8 钓鱼与底层机制

- `Fishing` 是高价值支线系统
- `Luck` 是低可见但高影响的底层修正系统
- 这些内容不应被归类成“可有可无的杂项”

## 3. 适合 TerraPedia 的一级内容域

最适合做一级导航或一级数据域的内容有：

1. 物品
2. 配方
3. 群系
4. Boss
5. 事件
6. NPC
7. 阶段推进

## 4. 对项目的直接启示

如果 TerraPedia 只做物品页，它只会是一个半成品。

如果 TerraPedia 把下面这些一起结构化：

- `items`
- `recipes`
- `biomes`
- `bosses`
- `events`
- `npcs`
- `progression`

它才真正具备“泰拉瑞亚系统百科”的骨架。
