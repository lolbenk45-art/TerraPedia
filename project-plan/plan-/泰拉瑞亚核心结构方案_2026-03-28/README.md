# 泰拉瑞亚核心结构方案

这套目录把 Terraria 的核心内容，按 TerraPedia 项目的落地视角整理成一组可直接使用的文档。

目标不是单纯介绍游戏，而是回答三个问题：

- Terraria 的核心系统到底有哪些
- 如果做 TerraPedia，信息架构该怎么组织
- 如果落到 MySQL，核心表和关系该怎么建

## 目录

- `01_泰拉瑞亚核心内容总纲.md`
- `02_产品与数据库建模.md`
- `03_前端信息架构与页面规划.md`
- `04_MySQL表结构草案.md`
- `05_实施优先级与里程碑.md`
- `06_API接口设计草案.md`
- `07_后台管理端字段与表单设计.md`

## 使用方式

- 如果你要先做内容理解：先看 `01`
- 如果你要做产品设计：先看 `03`
- 如果你要做后端与数据库：先看 `02` 和 `04`
- 如果你要排实施顺序：看 `05`

## 总结

这套方案的核心思想是：

- `items` 是主实体
- `recipes / sources / images / biomes / bosses / events / npcs` 是关系网络
- `progression` 是阶段推进框架
- `snapshots` 是来源与回溯层

这样建，TerraPedia 才会是“可检索的系统化百科”，而不是“堆页面的静态站”。
