# 实施优先级与里程碑

## 1. 第一阶段：最小可用闭环

优先做：

- `items`
- `recipes`
- `recipe_ingredients`
- `recipe_stations`
- `item_sources`
- `item_images`
- `biomes`
- `item_biomes`

完成后用户已经能解决最常见的问题：

- 这是什么物品
- 怎么做
- 怎么获得
- 在哪里找
- 长什么样

## 2. 第二阶段：推进系统成型

继续做：

- `bosses`
- `boss_drops`
- `events`
- `event_drops`
- `npcs`
- `npc_shops`
- `progression_stages`

完成后 TerraPedia 才会从“物品数据库”升级成“游戏推进百科”。

## 3. 第三阶段：增强层

再做：

- `snapshots`
- `luck`
- 更完整的战斗规则
- 多语言名称映射
- 版本差异比较

## 4. 前后端实施顺序建议

### 后端先做

1. `items`
2. `recipes`
3. `item_sources`
4. `item_images`
5. `biomes`

### 前端同步做

1. 物品列表页
2. 物品详情页
3. 配方展示模块
4. 获取来源模块
5. 群系关联模块

### 第二轮再做

1. Boss 列表与详情
2. NPC 列表与详情
3. Event 列表与详情
4. 阶段推进页

## 5. 交付判断标准

第一阶段算完成，至少应满足：

- 可以搜索物品
- 可以查看配方
- 可以查看来源
- 可以展示图片
- 可以从物品跳到相关 biome

第二阶段算完成，至少应满足：

- 可以按阶段浏览内容
- 可以查看 Boss 掉落和召唤条件
- 可以查看 NPC 商店和功能
- 可以查看事件产出

## 6. 最后结论

最稳的路线不是“一次建全”，而是按闭环递进：

1. 先把 `item -> recipe -> source -> biome -> image` 跑通
2. 再补 `boss / npc / event / progression`
3. 最后再做快照、版本、多语言和高级规则
