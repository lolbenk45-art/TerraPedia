## 3. 功能改动（已落地）

### 3.1 前端新增“写作后直接发布”
- 文件：
  - `data-query-app/pages/articles.vue`
  - `data-query-app/stores/articles.ts`
- 行为：
  - 编辑弹窗新增双提交意图：`Save Draft` / `Publish Now`
  - `Publish Now` 流程：
    1. 先保存（新建时先创建 DRAFT，编辑时先更新）
    2. 再调用 `PATCH /admin/articles/{id}/status`，传 `PUBLISHED`
  - 防重复提交与更清晰加载文案：
    - `Saving draft...`
    - `Publishing...`
  - 保留原有审核流按钮（送审/审核/发布/下线）不破坏

### 3.2 UI 优化（不改业务语义）
- 文件：`data-query-app/pages/articles.vue`
- 优化点：
  - `Submit Review` -> `Send for Review`
  - `Publish` -> `Publish Now`
  - 发布按钮视觉优先级提升
  - 编辑区加入流程提示文案

## 4. 后端能力校验结论
- 当前后端已支持前端一键发布链路，无需新增后端代码。
- 依据：
  - `PATCH /admin/articles/{id}/status` 支持 `PUBLISHED` 路径
  - 非严格模式下允许兼容提升到 `APPROVED` 后发布
  - 默认配置 `TERRAPEDIA_ARTICLE_REVIEW_ENFORCE_STRICT=false`
