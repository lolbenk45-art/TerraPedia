# TerraPedia M8-R1 后端数据时效入口统一执行计划
日期：2026-04-22  
对应里程碑：`M8-R1`

## 目标

先收掉后端数据时效链路中最基础、最容易反复污染后续工作的公共问题：

1. 数据脚本默认 API 基址仍残留 `8888`
2. 默认运行配置没有统一从本地栈配置读取
3. 后续统一后端刷新入口缺少稳定基础

## 本批范围

- `scripts/lib/local-runtime-config.mjs`
- 依赖默认 API 基址的数据脚本
- 验证脚本
- 执行与问题汇总文档

## 本批不做

- 不直接重构所有数据 pipeline
- 不一次性把所有域接进统一调度器
- 不进入前台相关能力

## 执行顺序

1. 先补本地运行配置的后端 API 基址解析能力
2. 再批量替换脚本里的历史 `8888` 默认值
3. 再跑脚本级最小验证
4. 若发现阻塞，沉淀到问题汇总文档

## 验收标准

- 数据脚本默认走当前本地后端端口 `18088`
- 仍允许显式 CLI / 环境变量覆盖
- 至少有一条测试覆盖新解析逻辑
- 修改后的脚本可通过最小验证
