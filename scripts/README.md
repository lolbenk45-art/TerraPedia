# Script Layout

- `scripts/dev`
  - Local stack entrypoints, local config templates, and verification probes
- `scripts/data`
  - Data collection, normalization, import, sync, backfill, and audit scripts
- `scripts/lib`
  - Shared local runtime helpers used across script groups
- `scripts/tooling`
  - Workspace maintenance and upstream monitoring helpers

Use the repo root as the working directory unless a script says otherwise.
